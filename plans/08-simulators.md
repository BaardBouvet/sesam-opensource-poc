# Plan 08: Simulated Source Systems

Build fake HubSpot and Tripletex APIs so we can develop and test without real accounts. Must support configurable jitter and failure injection.

## Options

### Option A: FastAPI Mock Servers — Recommended

- **What:** Build lightweight Python FastAPI apps that implement the subset of HubSpot/Tripletex APIs we use. Backed by in-memory data or SQLite.
- **Pros:**
  - Same language as the rest of the stack
  - Full control over behavior, jitter, and failure modes
  - Can generate realistic test data with Faker
  - Easy to containerize and use in testcontainers
  - OpenAPI spec auto-generated by FastAPI
- **Cons:**
  - Must manually implement each endpoint we use
  - Risk of simulator drift from real API behavior
- **Features:**
  - Configurable response latency (jitter)
  - Configurable error rates (429 rate limit, 500 server error, timeout)
  - Seed data generation (N contacts, M companies)
  - State: changes persist in memory during test run
  - Admin endpoints: `POST /admin/reset`, `POST /admin/configure-jitter`
- **Effort:** Medium

### Option B: WireMock / Prism

- **What:** Use API mocking tools that can replay recorded responses or generate from OpenAPI specs.
- **Pros:**
  - Can record real API traffic and replay it
  - Prism can generate mocks directly from OpenAPI specs
  - No custom code for basic mocking
- **Cons:**
  - Limited stateful behavior (WireMock stateful scenarios are clunky)
  - Harder to implement custom jitter/failure injection
  - Tripletex may not have a public OpenAPI spec
  - Java-based (WireMock) — different from rest of stack
- **Effort:** Low for basic mocking, High for stateful + failure injection

### Option C: Testcontainers + Recorded Fixtures

- **What:** Use pre-recorded JSON fixtures loaded into a generic mock server at test time via testcontainers.
- **Pros:**
  - Very fast test startup
  - Deterministic — same data every time
- **Cons:**
  - No stateful behavior (can't test write-back)
  - Fixtures go stale
  - Not useful for development, only unit/integration tests
- **Effort:** Low

## Simulator Scope

| Endpoint                        | HubSpot          | Tripletex        |
|---------------------------------|------------------|------------------|
| List contacts/customers         | GET /crm/v3/...  | GET /v2/...      |
| Get single contact/customer     | GET /crm/v3/...  | GET /v2/...      |
| Create contact/customer         | POST             | POST             |
| Update contact/customer         | PATCH            | PUT              |
| List companies                  | GET /crm/v3/...  | GET /v2/...      |
| Create/Update company           | POST/PATCH       | POST/PUT         |
| Search (if needed)              | POST /search     | GET with filters |
